BestGuide Case Study

Operations systems for tutoring and office workflows.

Web applications supporting office administration, teacher matching, student workflow management, part-time staff activity tracking, and supporting company web presence.

Back to portfolio View resume Contact me
  • Operations focus staff, teachers, and office workflows
  • Full web apps C# .NET MVC with SQL Server
  • Business support search, status, balances, and approvals

Overview

The systems in this track were designed to reduce administrative friction for a tutoring business. The work touched order handling, teacher matching, staff activity tracking, financial visibility, and role-based access to operational data.

These were workflow-heavy products where usability and process clarity mattered as much as the code itself.

Core stack

  • HTML
  • CSS
  • Bootstrap
  • C# .NET MVC
  • Microsoft SQL Server
Office system Operational management

Office Management System

Internal system built to streamline the company’s daily operations, manage requests, and improve task and order visibility for staff.

  • Supported request intake, recording, categorization, and status tracking.
  • Matched students with suitable teachers based on defined criteria.
  • Presented matching information so students could choose teachers.
  • Tracked order progress across pending and completed stages.
  • Enabled task-management visibility for officers and daily assignments.
  • Supported financial indicators for monitoring budgets, expenses, and payments.
  • Enforced role-based access control across system features.
Tutoring website Teacher and student matching

Tutoring website

Public-facing teaching workflow where teachers could search for students based on location and submit requests tied back to office operations.

  • Implemented location-based search so teachers could find suitable students.
  • Used Office Management System data for classes and locations.
  • Allowed teachers to submit interest requests through the platform.
  • Supported approval-based progression before teaching could begin.
Staff portal Daily activity and balances

Staff portal for part-time team members

Web application for part-time staff to submit daily work, track balances, and initiate withdrawal after office approval.

  • Built submission flows for daily activities performed by part-time staff.
  • Applied pricing rules to submitted work items.
  • Supported office review and approval through the Office Management System.
  • Updated balances when approved work increased a staff member’s amount.
  • Enabled staff to review balances and request withdrawals.
Company website Information and registration flows

Static company website

Supporting site for company information, registration flows, and teacher applications, designed for clarity and accessibility.

  • Built a company introduction covering mission, values, and services.
  • Supported student registration for offered lessons.
  • Created a section for teacher job applications.
  • Maintained the site as a simple public information layer for the business.
